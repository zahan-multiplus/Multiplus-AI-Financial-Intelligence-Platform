## Summary: Key Reuse Opportunities

- **Reuse as-is (do not replace with AI):** User risk score formula, questionnaire composite score and bands, fund risk score, asset allocation rules, allocation validation, fund matching by risk gap, NAV ingestion and metrics pipeline, document upload/storage, product catalog, CRM/MFAPI/Kyckart integrations, MySQL/MongoDB/S3 storage, notification delivery (SMS/email/WhatsApp).
- **AI consumes output:** Risk scores (user and fund), allocation and matching funds, portfolio and fund metrics, stored documents, product data. Use for explanations, insights, recommendation reasoning, and optional anomaly detection.
- **Extend with AI:** (1) **Explanations:** Backend calls AI module with existing outputs (risk score, allocation, matching funds) to generate natural-language explanations. (2) **Risk profile:** Use AI to generate narrative insights or report text from questionnaire results. (3) **Document parsing:** Keep credit report PDF pipeline; add AI for harder layouts or new doc types (CAS, bank, insurance) via a new document-extraction component. (4) **Notifications:** When content is AI-generated, backend calls AI then uses existing SmsService/Mailer/WhatsApp to send.
- **New AI components:** (1) **Model serving** for LLM (and optionally embeddings). (2) **Explanation service** that takes structured context (score, allocation, profile) and returns text. (3) **Document intelligence** for non–credit-report docs (OCR + extraction), writing into existing storage. (4) Optional: **anomaly detection** or **insight pipeline** that reads from existing analytics outputs and raises alerts or insights.
- **Integration patterns:** **Service calls AI module** for explanations and notification content; **AI reads from service output or data store** for all consumed data; **event-driven** optional for triggering AI on RiskScoreComputedEvent. Deterministic rules (allocation, suitability, scoring) remain in existing services; AI augments with explanation and insight only.
